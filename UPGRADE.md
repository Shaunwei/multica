# Multica Upgrade — see the canonical skill

The full upgrade procedure for Multica (worker nodes and the Air full-stack host) is maintained as a skill in Shaun's private skills repo:

- **Canonical:** `Shaunwei/skills` → `work/multica-upgrade/SKILL.md`
- **Synced into:** `~/.openclaw/workspace/skills/work/multica-upgrade/SKILL.md` on each machine via `bootstrap/sync-skills.sh`

Read it there. Do not edit this file — patches must land in the skills repo so all machines get them on the next sync.

## Quick map

- **Air** (control plane): full stack — CLI + Server + Web UI + Daemon, plus `make migrate-up`
- **Pro / Max** (workers): CLI + Daemon only, pulled from `fork/deploy-private`

## If the skill repo is not synced on this host

Fetch the SKILL.md directly:

```bash
gh repo clone Shaunwei/skills /tmp/skills 2>/dev/null
cat /tmp/skills/work/multica-upgrade/SKILL.md
```
